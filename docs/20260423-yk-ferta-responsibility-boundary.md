# yk-FERTA 责任边界（简版）

## 1. yk-FERTA 智能体团队负责

- 打磨并交付诊断推理工作流内核  
  `HPO确认 -> 候选生成 -> 证据检索 -> 逐病复核 -> 最终综合`
- 定义并稳定后端接口（REST + SSE）
- 输出结构化结果与可追溯中间产物（artifacts）
- 提供最小前端交互页面（用于流程验证与对接演示）
- 提供最小任务存储（临时数据库）以支撑任务执行与回放

## 2. IT / 平台团队负责

- 统一登录、授权、鉴权、组织权限体系
- 历史病例、个人记录、长期数据治理与归档
- 平台级审计、监控告警、网关与发布体系
- 最终系统中的 UI 融合、样式改造与业务集成

## 3. 本阶段交付目标（yk-FERTA）

- 一个可嵌入公司系统的“智能体推理服务”
- 一套稳定的接口契约（字段、状态、错误语义）
- 一套可运行的最小演示前端（非最终产品 UI）

## 4. 非目标（本阶段不做）

- 不做账号体系与权限管理
- 不做个人中心/历史中心等平台功能
- 不做完整生产级数据平台与运维体系
- 不做最终产品形态 UI 定稿

## 5. 对接原则

- yk-FERTA 负责“诊断推理能力与接口标准化”
- IT 负责“系统级集成与平台化能力”
- 双方以接口契约和字段版本为边界协作
